## 2. 운영 특성 분석 결과

**핵심: 부하의 시간 구조가 없다 (상시 균일 가동).**

- 전 설비 24시간·전 요일 균일 가동. 시간대별 평균 부하 변동 < 0.1%, 주말/야간 감소 없음 (`02_요일x시간대_히트맵.png`)
- 설비별 평균 전력 모두 ~3,010W, 총 전력량도 13개 설비 모두 10.83MWh(±0.05%)로 사실상 동일 (`01_설비별_총전력량.png`)
- 유효전력 분포도 13개 설비가 동일 (중앙값 3,009~3,011W, p1~p99: 1,420~4,600W)
- operation 플래그는 전 구간 1 (가동/비가동 구분 정보 없음)
- 월별 추이도 완전 평탄 (계절 효과 없음)

→ **시간대·요일 기반 부하 분석(피크 시프팅, 대기전력 등)으로는 비효율을 찾을 수 없는 데이터.**
→ 비효율 신호는 시간 축이 아니라 **설비 축 + 전력 품질 지표**에 심어져 있음.

- 나머지 10개 설비에서는 이상 0건

## 4. 리포트 스토리라인 시사점

1. **운영 특성 분석**: "13개 설비가 24시간 균일 부하로 상시 가동되는 연속 공정. 부하 총량 관점의 낭비는 없음" → 그래서 전력 *양*이 아닌 전력 *품질*로 관점 전환하는 논리
2. **비효율 정의**:
   - 저역률 운전 = 동일 유효전력 전달에 더 큰 피상전력/전류 필요 → 선로 손실 증가, 한전 역률 요금 페널티(지상역률 90% 미달 시 60%까지 0.2%/1%p 추가요금) 근거
   - 전압 강하 = 설비 오작동·수명 단축·품질 불량 리스크, 접촉 불량/과부하 징후
3. **탐지 모델**: 라벨이 없으므로 ① 물리 규칙 기반 기준(전압<210V, 역률<85%)으로 준거 라벨 생성 → ② 비지도 모델(Isolation Forest, 설비별 다변량)로 탐지 → ③ 규칙 준거 대비 Precision/Recall 평가 구조가 성립
4. **개선 효과 정량화**: 저역률 이벤트의 무효전력 증분 → 역률 개선 시 절감 가능 피상전력·요금, 전압 이상 조기 경보의 다운타임 회피 가치로 환산
5. **AI 관리 모델 제안**: 5초 스트림 실시간 이상 탐지 + 설비 건강도 스코어링 + 콘덴서 뱅크 제어/정비 알림으로 연결

## 5. 주의할 점 (리포트 작성 시)

- 공모전 요강 문구는 "데이터 센터"지만 실제 데이터·노션 공지는 **제조공장(건조·분쇄 공정)** — 제조공장으로 프레이밍할 것
- 시간대 패턴이 없다는 사실 자체를 "분석 결과"로 정직하게 제시하고 관점 전환의 근거로 사용 (억지로 피크 스토리를 만들지 말 것)
- 이상이 5초 단발성이므로 "구간" 탐지를 위해 윈도우 단위(예: 1시간 내 이벤트 빈도) 집계 지표 설계가 필요
